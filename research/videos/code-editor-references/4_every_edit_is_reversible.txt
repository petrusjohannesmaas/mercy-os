## Summary
- Claude Code ensures **file safety with checkpoints**: before any edit, it snapshots the file so changes are reversible. Pressing escape twice restores the file to its prior state.  
- **Checkpoints apply only to local file changes**. Actions affecting remote systems (databases, APIs, deployments) cannot be undone, so Claude asks for confirmation before running commands with external side effects.  
- Claude provides **three permission modes**:  
  - **Default mode**: asks before both edits and commands.  
  - **Autoedit mode**: allows file edits freely, but still asks before commands.  
  - **Plan mode**: read‑only; Claude analyzes and creates a plan without making changes until approved.  
- Trusted commands can be **allow‑listed** in `settings.json`. This can be configured per project (`.cloud/settings.json`), per user (home config), or organization‑wide.  
- The combination of **checkpoints, permission modes, and allow lists** gives developers control, reversibility, and speed while maintaining safety.  

This defines Claude Code’s **safety model**: reversible file edits, explicit confirmation for external actions, and configurable permissions for balanced control and efficiency.
