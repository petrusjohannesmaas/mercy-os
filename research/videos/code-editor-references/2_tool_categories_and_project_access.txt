## Summary
- Claude Code has **five categories of built‑in tools**:  
  - **File operations**: read, edit, create files  
  - **Search**: find files by pattern, search content with regex  
  - **Execution**: run shell commands, start servers, run tests, use git  
  - **Web access**: search the web, fetch documentation  
  - **Code intelligence**: see type errors, jump to definitions, find references  

- These tools are **chained together** in the agentic loop: run tests → read errors → search files → read files → edit code → rerun tests. Each tool call informs the next decision.  

- Claude has **full access to your project environment**:  
  - Entire project files in the working directory  
  - Full terminal access for any shell command  
  - Git state, branches, changes, history  
  - `claude.md` file for project‑specific instructions  
  - Automemory across sessions  

- **claude.md** acts as a persistent prompt: coding conventions, test commands, architecture decisions, shared via version control.  

- Claude Code can be **extended beyond built‑in tools** in four ways:  
  - **Skills**: reusable prompt files for workflows  
  - **MCP servers**: connect external tools and data sources  
  - **Hooks**: run custom scripts before/after tool calls  
  - **Sub‑agents**: spawn specialized agents for focused tasks  

- In practice, Claude can:  
  - Search patterns across multiple files  
  - Read several files to understand architecture  
  - Make coordinated edits across components  
  - Run tests to verify everything works together  

- Core takeaway: Claude Code combines five tool categories, full project access, and extensibility through skills, MCP, hooks, and sub‑agents, enabling it to understand and modify your entire codebase as a connected system.
