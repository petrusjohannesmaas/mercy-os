## Summary
- Each run of Claude in the terminal starts a **fresh session** with no conversation history carried over.  
- Claude can persist knowledge across sessions using **autogenerated memory** and **claude.md** files.  
- You can **resume sessions** with `claude d-continue`, **browse past sessions** with `claude-resume`, and **fork sessions** with `claude d-continue-fork` (like git branching for conversations).  
- The **context window** fills with different elements:  
  - Conversation messages (~45%)  
  - File contents Claude has read (~22%)  
  - System instructions (~15%)  
  - claude.md files (~10%)  
  - MCP tool definitions (~8%)  
- When the context window fills, Claude **compacts automatically**, summarizing older tool results while keeping important parts. You can also trigger `/compact` manually.  
- Persistent rules and instructions should go in **claude.md**, not in conversation, since conversation content gets compacted away.  
- **Sub‑agents** help manage context during large tasks. Each sub‑agent gets its own fresh context window, does its work, and returns a summary to the main session, keeping the main context small and focused.  
- Core takeaway: sessions are independent but can be resumed or forked, the context window is actively managed through compaction, and sub‑agents keep tasks organized without overwhelming the main session.  

This defines Claude Code’s **sessions and context model**: fresh starts each run, resumable and forkable sessions, structured context management, and sub‑agents for scalability.
