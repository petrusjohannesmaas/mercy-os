Here’s the **balanced consolidation of the core agentic loop** — blending the Claude Code docs (loop, tools, safety, sessions) with Anti‑Gravity’s artifact system, and using the **hidden folder approach** for persistence:

---

## Core Agentic Loop with Artifact Persistence

- **Gather context**  
  - Agent investigates: runs tests, reads errors, searches files, loads docs.  
  - Produces a **Task List artifact** (markdown file in `.agentic/`): records what it has learned and outlines next steps.  
  - Stored locally in the hidden folder, versionable with git, ensuring transparency and persistence across sessions.  

- **Take action**  
  - Agent edits files, runs commands, generates code.  
  - Produces an **Implementation Plan artifact** (markdown in `.agentic/`): details intended changes, rationale, and verification strategy before modifying the codebase.  
  - Acts as a checkpoint for human review before execution.  

- **Verify results**  
  - Agent re‑runs tests, checks outputs, or uses browser automation.  
  - Produces a **Walkthrough artifact** (markdown in `.agentic/`): documents completed work, verification steps, and evidence (logs, screenshots, PR descriptions).  
  - Serves as proof of correctness and a record for future debugging.  

- **Loop until completion**  
  - Each cycle generates or updates artifacts.  
  - Artifacts persist in the hidden folder, keeping reasoning, plans, and verification accessible across sessions without bloating the context window.  

---

## Barebones Agentic IDE Features (balanced)

1. **Agentic loop**: context → action → verification, repeated until done.  
2. **Artifact system**: task list, implementation plan, walkthrough — stored in `.agentic/` as markdown for transparency and persistence.  
3. **Tool categories**: file operations, search, execution, web access, code intelligence.  
4. **Session/context management**: fresh sessions, resumable/forkable, compaction, sub‑agents.  
5. **Safety controls**: checkpoints for file edits, permission modes, allow lists for trusted commands.  
6. **Prompting discipline**: verification criteria, scoped prompts, rich content, persistent project rules (`claude.md` equivalent).  

---

**Core takeaway**: By combining Claude Code’s structured loop and safety model with Anti‑Gravity’s artifact system — and persisting artifacts in a hidden `.agentic/` folder — you get a barebones agentic IDE that is transparent, inspectable, and resilient across sessions. The artifacts act as living documentation of context, plan, and verification, while the loop ensures iterative progress.
